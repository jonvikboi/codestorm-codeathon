Team assignments

P1
Person 1 — Frontend Lead
Next.js · Tailwind · Shadcn · Dashboard
Frontend
  • Setup: Initialize Next.js project template, configure project folders (frontend/backend split), wire up base Tailwind & Tailwind CSS configuration
  • Template Core: Establish basic layout, dashboard shell, navigation framework, and theme context config to serve as a shared boilerplate
  • Boilerplate UI: Set up a skeleton page structure and mock API router configuration so teammates have a reference template
  • Documentation: Document repo onboarding, project setup instructions, and component usage guide for the rest of the team
P2
Person 2 — Frontend + UI Polish
Next.js · Task CRUD · Attendance UI · Pitch
Frontend AI UI
  • Core: Task/Deadline CRUD UI — form with WhatsApp reminder time picker + "Add to Calendar" toggle, task list with status badges
  • Module — Attendance Risk Alerter UI: Subject attendance % inputs, at-risk subjects highlighted, classes-needed calculation display, weekly alert preview
  • Integration: Wire both modules to backend API (Axios calls), handle loading/error states, show n8n trigger confirmation toasts
  • Demo prep: Pitch deck (2 min), demo script, seed realistic attendance data, run full user journey walkthrough with team
P3
Person 3 — Backend + AI API
Node.js/Express · Supabase · Groq API
Backend AI API
  • Setup: Init Express project, Supabase schema (students, tasks, attendance tables), auth with JWT, .env for Groq + Supabase + n8n keys
  • Core: POST /register, POST /login, full CRUD for /tasks — on task create, immediately POST to n8n deadline webhook with student phone + deadline details
  • AI endpoints: POST /ai/study-schedule (deadline + subjects → Groq → day-wise study plan JSON), POST /ai/attendance (subject % array → Groq → risk flags + min classes needed)
  • Polish: GET /automations (fetch n8n execution logs), robust error handling + input validation, README with all env vars and setup steps
P4
Person 4 — n8n + Integrations
n8n Cloud · Twilio WhatsApp · Google Calendar
n8n Backend
  • Setup: n8n Cloud account + 2 blank workflows, Twilio WhatsApp sandbox (5 min at twilio.com), Google Calendar OAuth2 — verify all three with a dummy POST before touching anything else
  • Workflow 1 — Deadline Reminder: Webhook → Set (format ISO date, compute reminderTime) → Google Calendar (create event) → Wait (24hr before) → Twilio WhatsApp (reminder) → Wait (1hr before) → WhatsApp (final nudge)
  • Workflow 2 — Attendance Alert: Webhook → Set (weekly trigger, build at-risk list) → Twilio WhatsApp (send alert per student: "You need X more classes in DBMS!")
  • Testing: End-to-end test both workflows from the UI — confirm WhatsApp arrives on real phone + Calendar event appears — share Production webhook URLs with P3
